Reques 3:
Risk1: Scope Creep
Description: Scope Creep occurs when stakeholders changes resquest without control and evaluation
Possible impacts:
 - Quality: Descrease quality due to overloads tasks
 - Time: Project timeline delay
 - Cost: Increasing cost budget due to extra resource or extended duration
Risk response plans:
 - Mitigation: clearly define and document of the project scope at the beginning
 - Contingency: Allocated buffer time and budget for potential changes

Risk 2: Key developer resignation
Description: Key developer may leave the project, causing loss of critical knowledge
Possible impacts:
 - Quality: Descrease quality due to 
 - Time: Delay while hiring a replacement
 - Cost: Increasing by hire a replacement or training new personnel
Risk response plans:
 - Mitigation: Build document project to support knowledge transfer better, sharing knowledge and cross-training within the team
 - Contingency: Quickly finding a replacement

Risk 3: Third-party API downtime
Description: The project depend on service API, when API become unavaiables, system functional problem
Possible impacts:
 - Quality: System feature may not function properly
 - Time: delay while waiting APIs recovery or fixing
 - Cost: Addition cost for technical fixes
Risk response plans:
 - Mitigation: Backup hosting local information of customers, database or system project
 - Contingency: Switch to backup API
